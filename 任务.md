### 任务概述

基于免费公开 API，使用 AI 工具构建一个**智能食谱助手**。用户通过对话描述想吃什么，Agent 搜索菜谱和配饮并给出推荐。

### 可用 API

以下 API 全部免费，无需注册，无需 API Key，可直接调用。

### TheMealDB — 菜谱数据

文档：https://www.themealdb.com/api.php

```
# 按名称搜索菜谱
GET <https://www.themealdb.com/api/json/v1/1/search.php?s={name}>

# 按食材筛选
GET <https://www.themealdb.com/api/json/v1/1/filter.php?i={ingredient}>

# 按分类筛选
GET <https://www.themealdb.com/api/json/v1/1/filter.php?c={category}>

# 按地区筛选
GET <https://www.themealdb.com/api/json/v1/1/filter.php?a={area}>

# 按 ID 查看详情
GET <https://www.themealdb.com/api/json/v1/1/lookup.php?i={id}>

# 随机推荐
GET <https://www.themealdb.com/api/json/v1/1/random.php>

# 所有分类
GET <https://www.themealdb.com/api/json/v1/1/categories.php>
```

### 必须完成

### 1. AI Agent（后端）

- 使用 OpenRouter API 调用大语言模型
- 实现**至少 3 个 Tool / Function**：
  - `search_meals` — 按关键词或食材搜索菜谱
  - `get_meal_detail` — 获取菜谱详细信息（食材清单、烹饪步骤）
  - `search_cocktails` — 搜索饮品/鸡尾酒
- Agent 能根据用户自然语言输入，**自主决定**调用哪个 Tool
- 支持多轮对话

### 2. Web 界面（前端）

- Chat 对话界面（用户输入 → Agent 回复）
- 搜索结果需**结构化展示**（菜谱卡片含图片、食材列表等），不能只显示原始 JSON
- ui基础交互可用

### 加分项

- 餐食搭配推荐（"给我推荐一顿晚餐" → 主菜 + 配饮组合）
- 按饮食偏好过滤（素食、某地区菜系等）
- Agent 思考过程可视化（展示 Tool 调用链路）
- 流式输出（Streaming）
- 完善的错误处理与优雅降级
- 精美加分
