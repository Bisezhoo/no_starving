🔍 跨接口关键发现

1. filter 接口只返回摘要
TheMealDB filter 返回 5 字段，TheCocktailDB filter 返回 3 字段。需配合 lookup 获取完整数据。
2. strCountry 零缺失
在所有 TheMealDB 接口中，strCountry 始终有值（0% 缺失），而 strArea 在 filter_i/filter_c 中有 ~10% 缺失。推荐用 strCountry。
3. 无结果返回格式不一致
TheCocktailDB filter 无结果返回 "no data found" 字符串，其他接口返回 null。
4. TheCocktailDB 中文翻译未实现
strInstructionsZH-HANS 和 strInstructionsZH-HANT 字段存在但 99~100% 为 null。
5. 永远为空的字段（6 个）
TheMealDB: strMealAlternate、strImageSource、strCreativeCommonsConfirmed
TheCocktailDB: strDrinkAlternate、strInstructionsZH-HANS、strInstructionsZH-HANT
6. TheCocktailDB 特有字段
strAlcoholic（含酒精类型，0% 缺失）、strGlass（杯型，0%）、strIBA（IBA 分类，75~86% 缺失）
7. 食材槽位差异
TheMealDB 最多 20 种食材，TheCocktailDB 最多 15 种。实际常用上限：MealDB ~12，Cocktail ~6。
8. 空搜索行为
TheMealDB search.php?s= 返回 25 条默认数据（可当推荐接口），TheCocktailDB 返回 null。
9. 参数大小写
两个 API 的 filter 参数均不区分大小写。
