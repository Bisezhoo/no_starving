## 1. 背景

`AssistantResultBlock.vue` 原先会把本轮 Agent 返回的全部推荐卡片直接平铺出来。推荐数量较多时，结果区会明显拉长页面，也会让用户在继续查看步骤或后续对话前产生额外滚动成本。

本次需求要求：最终推荐结果默认只展示 3 个，其余隐藏；点击“查看更多”后展示全部，但不能继续纵向平铺，而要使用横向滑动轨道；同一个按钮变为“收起”，点击后恢复默认 3 个。

## 2. 目标

1. `message.cards.length <= 3` 时展示全部卡片，不显示按钮。
2. `message.cards.length > 3` 时默认只展示前 3 个卡片。
3. 默认态按钮文案为“查看更多”，点击后进入展开态。
4. 展开态展示全部卡片，并把卡片容器切换为横向可滚动轨道。
5. 展开态按钮文案变为“收起”，点击后恢复默认 3 个卡片。
6. 食材/步骤辅助区与当前可见卡片保持一致，隐藏卡片的步骤不提前露出。

## 3. 非目标

1. 不修改后端推荐数量、排序、去重或 SSE 事件契约。
2. 不新增分页或加载更多接口。
3. 不引入第三方轮播库。
4. 不实现自动轮播、分页圆点或左右箭头。

## 4. 实现设计

采用前端展示层最小改造：

1. `AssistantResultBlock.vue` 新增本地状态 `isExpanded`。
2. 新增 `DEFAULT_VISIBLE_CARD_COUNT = 3`。
3. 通过 `visibleCards` 统一驱动卡片区和步骤区：
   - 默认态：`message.cards.slice(0, 3)`。
   - 展开态：`message.cards`。
4. 卡片渲染按 `visibleCards` 顺序执行，再根据 `card.type` 分发到 `MealCard.vue` 或 `CocktailCard.vue`，保持后端返回顺序。
5. `hasOverflowCards` 为 `message.cards.length > 3` 时才展示“查看更多/收起”按钮。
6. 展开态给卡片容器增加 `card-slider` class；CSS 使用原生 `overflow-x: auto`、`display: flex` 和 `scroll-snap-type` 实现横向滑块。

## 5. 交互状态

| 状态 | 卡片数量 | 容器形态 | 按钮 |
|------|----------|----------|------|
| 无溢出 | 全部，最多 3 个 | 原网格 | 不显示 |
| 默认收起 | 前 3 个 | 原网格 | 查看更多 |
| 展开全部 | 全部 | 横向滑动轨道 | 收起 |

## 6. 测试方案与结果

| 测试 | 断言 | 结果 |
|------|------|------|
| 3 个以内推荐 | 渲染全部卡片；不显示“查看更多”按钮 | ✅ 通过 |
| 超过 3 个推荐默认态 | 只渲染前 3 个卡片；第 4 个卡片和步骤不出现；按钮为“查看更多” | ✅ 通过 |
| 点击查看更多 | 全部卡片出现；容器进入横向滑动模式；按钮变为“收起” | ✅ 通过 |
| 点击收起 | 恢复只显示前 3 个卡片；第 4 个卡片隐藏；按钮恢复“查看更多” | ✅ 通过 |

## 8. 方案取舍

推荐使用组件内状态和原生横向滚动，因为这次需求只影响前端展示方式。让后端只返回 3 个会破坏当前完整 `cards` 返回契约；新增分页接口会引入状态同步和幂等问题；第三方轮播库会增加依赖和测试复杂度。原生横向滚动足够满足“展示全部但不平铺”的目标。
