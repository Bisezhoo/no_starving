## 1. 背景

当前推荐结果区会在卡片下方直接渲染当前可见卡片的食材、用量和步骤。菜谱详情较长时，页面会被连续的详情列表撑高，用户需要大量滚动才能继续看后续对话。

本次变更要求把“查看单个菜谱详情”改为显式动作：卡片内增加“查看详情”按钮，用户点击后通过弹窗查看当前卡片的完整详情。为保持结构化卡片体验一致，饮品卡片也使用同一套详情弹窗。

## 2. 目标

1. 每张菜谱和饮品卡片都展示“查看详情”按钮。
2. 点击按钮后只打开当前卡片的详情弹窗，不影响其他卡片。
3. 弹窗展示标题、摘要、分类信息、食材/配料和做法步骤。
4. 做法步骤优先展示 `localizedInstructions`，没有本地化步骤时回退 `instructions`。
5. 推荐结果区不再在卡片下方直接平铺食材和步骤。
6. 保持已有“默认 3 个推荐 / 查看更多横向滑块 / 收起”交互不变。

## 3. 非目标

1. 不修改后端 Tool、SSE 事件、`Card[]` 数据契约或推荐排序。
2. 不新增详情接口；弹窗使用当前卡片已有的标准化字段。
3. 不引入第三方弹窗组件库。
4. 不实现卡片间上一条/下一条切换。
5. 不把详情弹窗状态持久化到聊天历史。

## 4. 实现设计

采用前端展示层最小改造：

1. `MealCard.vue` 和 `CocktailCard.vue` 增加 `emit("view-detail", card)`，按钮点击时把当前卡片交给父组件。
2. 卡片正文保留标题、摘要、分类、标签等概要信息，不再直接渲染完整 `ingredients` 列表，避免卡片本身变成长详情。
3. `AssistantResultBlock.vue` 持有 `selectedCard` 状态：
   - 初始为 `null`。
   - 收到卡片的 `view-detail` 事件时设置为当前卡片。
   - 点击关闭按钮或弹窗遮罩时恢复为 `null`。
4. `AssistantResultBlock.vue` 移除原 `hasSteps` 详情辅助区，改为根据 `selectedCard` 渲染单个弹窗。
5. 弹窗内部复用 `stepsFor(card)`，确保本地化步骤优先级与旧详情区一致。
6. CSS 使用原生固定定位遮罩和居中面板；移动端限制最大高度并允许弹窗内容滚动。

## 5. 交互状态

| 状态 | 触发 | 展示 |
|------|------|------|
| 默认态 | 有推荐卡片 | 卡片概要 + “查看详情”按钮 |
| 打开详情 | 点击某张卡片按钮 | 遮罩 + 当前卡片详情弹窗 |
| 关闭详情 | 点击“关闭”或遮罩 | 弹窗消失，推荐卡片状态不变 |
| 展开更多 | 点击“查看更多” | 保持原横向滑块；每张卡片仍可打开自己的详情 |

## 6. 测试方案

| 测试 | 断言 |
|------|------|
| 卡片按钮 | 菜谱卡片和饮品卡片都渲染“查看详情”按钮 |
| 默认不平铺详情 | 渲染推荐结果后，详情步骤不在结果区下方直接出现 |
| 打开弹窗 | 点击单张卡片按钮后，弹窗展示该卡片标题、食材和步骤 |
| 本地化优先 | 同时存在 `localizedInstructions` 和 `instructions` 时，弹窗展示本地化步骤，不展示英文步骤 |
| 关闭弹窗 | 点击关闭按钮后，弹窗从 DOM 中移除 |
| 横向滑块兼容 | 超过 3 张时默认仍只展示前 3 张，展开后第 4 张可打开自己的详情 |
